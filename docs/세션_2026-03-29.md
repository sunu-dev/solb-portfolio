## 스케일링 전략 (전략회의 결과)

### 핵심 원칙
"서버가 1번 호출하고, 사용자 N명이 캐시를 읽는 구조"

### 단계별 계획

| 단계 | 사용자 | 구조 | 비용 |
|------|--------|------|------|
| Phase 1 (현재) | ~100명 | Vercel API Route + Finnhub/Yahoo 직접 | 무료 |
| Phase 2 (500명+) | ~1,000명 | Vercel Cron → Upstash Redis 캐시 → API Route | 월 $25 |
| Phase 3 (5,000명+) | ~10,000명 | 전용 시세 서버 + WebSocket 브로드캐스트 | 월 $250 |

### 뉴스 파이프라인

| 단계 | 방법 |
|------|------|
| 즉시 | 뉴스 캐시 API Route (서버사이드 30분 캐시) |
| 중기 | Vercel Cron + Supabase 저장 (30분 주기 수집) |
| 장기 | NewsAPI.org 또는 자체 크롤러 |

### 비용 시뮬레이션 (1만 유저)
- Finnhub 프리미엄: $149/월
- Upstash Redis: $10/월
- Vercel Pro: $20/월
- Supabase Pro: $25/월
- Gemini 유료: $50/월
- **합계: ~$250/월 (35만원)**
- **수익: 4,900원 × 500명 유료 = 245만원/월 → 충분히 커버**
