## "이거 꼭 확인해야 돼?" — 정직한 답

- **지금(현 배포 상태)에서 법적으로 강제되어 "즉시 반드시" 해야 하는 건 없다.** 전부 ① 다음 단계 게이트(공개·과금·플래그 on) 또는 ② 사후 권장.
- **유일하게 '이미 라이브인데 미검토'인 건 L1(레버리지 약관 v4)** — 원래 문서엔 "배포 전 금지"였으나 파운더가 **선배포 결정**(분쟁 명백 행위 회피 + 회색지대 활용, 베타 규모·방어카드 충분)으로 이미 배포. 그래서 이제 **"배포 전 게이트"가 아니라 "사후 점검"**이고, 변호사 받으라는 법은 없으므로 **권장(분쟁 대비)**이지 필수가 아니다. → `DEPLOY_RISK_LEVERAGE.md §4`에 결정·사유 기록 완료.
- **세무**: v1 합산기 = **세무사 불필요**(증권사 숫자 합산만). 변호사는 **공개·과금 직전**(지금 프리뷰는 비공개라 불필요). 세무사 calc 감수는 **v2 자체계산** 때만.
- **digest·PRO**: 다음 단계(플래그 on / 결제 출시) 직전 게이트 → 지금 불필요.

## 권고 우선순위 (전문가 1회 상담으로 묶기)

세무 공개를 결정하는 시점에 **변호사 1회 상담으로 L1+L2+T1을 한 번에** 처리하는 게 비용 효율적:
- L1·L2 (레버리지 §6/§101 사후 점검) + T1 (세무 §20③·약관규제법 §7) → 자본시장법+핀테크 변호사 1명.
- 세무사는 v2 진입할 때만(그 전엔 불요).
- 질문지: 레버리지 `LEGAL_CONSULTATION_LEVERAGE.md`(11문) + 세무 `LEGAL_CONSULTATION_TAX.md` B섹션(6문).
