# SOLB PORTFOLIO — 카카오 로그인 설정 가이드 (완료본)

## 문제 상황

카카오 로그인 시 **KOE205 에러** 발생:
```
잘못된 요청 (KOE205)
설정하지 않은 카카오 로그인 동의 항목을 포함해 인가 코드를 요청했습니다.
설정하지 않은 동의 항목: account_email
```

## 원인

Supabase가 카카오 로그인 시 **`account_email` scope를 자동으로 요청**합니다.
하지만 카카오에서 이메일 동의항목은 **비즈 앱**만 사용 가능합니다.
개인 개발자 계정에서는 이메일 권한이 없어서 KOE205 에러가 발생합니다.

> **참고:** 코드에서 scope를 제거해도 Supabase 서버에서 강제로 추가하기 때문에
> 코드 수정만으로는 해결이 안 됩니다.

## 해결 과정

### 1단계: Kakao Developers 앱 생성

1. https://developers.kakao.com 접속 → 카카오 계정 로그인
2. **"앱 생성"** → 앱 이름: `SOLB PORTFOLIO`, 사업자명: 본인 이름
3. 생성된 앱 클릭

### 2단계: 앱 기본 정보 설정

1. 왼쪽 메뉴 → **앱 설정** → **일반**
2. 앱 기본 정보:
   - 앱 이름: `SOLB PORTFOLIO`
   - 카테고리: `금융`
   - 앱 대표 도메인: `https://solb-portfolio.vercel.app`

### 3단계: 앱 아이콘 등록 (비즈 앱 전환에 필요)

1. 앱 설정 → 일반 → **앱 아이콘** → 아이콘 업로드
2. 권장 사이즈: **128x128 px**, 최대 250KB
3. SOLB 로고 아이콘 사용 (파란 배경 + "SB" 텍스트)

### 4단계: 비즈 앱 전환 — 개인 개발자 등록

> **왜 필요한가?**
> 카카오 이메일 동의항목은 비즈 앱만 사용 가능합니다.
> 사업자등록증 없이도 **"개인 개발자"**로 비즈 앱 전환이 가능합니다.

1. 앱 설정 → 일반 → 하단 **"비즈니스 정보"** 섹션
2. **"비즈 앱 전환"** 또는 **"개인 개발자 등록"** 클릭
3. 본인 인증 완료
4. 비즈 앱 전환 완료

### 5단계: 카카오 로그인 활성화

1. 왼쪽 메뉴 → **카카오 로그인** → **일반**
2. **상태** 토글 → **ON**

### 6단계: Redirect URI 등록

1. 같은 페이지에서 **"카카오 로그인 리다이렉트 URI"**
2. 입력: `https://ytowigkzunbrhyucbsqu.supabase.co/auth/v1/callback`
3. 저장

> **이 주소가 뭔가요?**
> 사용자가 카카오 로그인을 마치면, 카카오가 이 주소로 "로그인 성공" 정보를 보냅니다.
> Supabase가 이걸 받아서 우리 앱에 전달합니다.

### 7단계: 동의항목 설정 (핵심!)

1. 왼쪽 메뉴 → **카카오 로그인** → **동의항목**
2. 각 항목 설정:

| 항목 | 상태 | 동의 목적 |
|------|------|----------|
| **닉네임** | ● 필수 동의 | `서비스 내 사용자 식별 및 표시` |
| **프로필 사진** | ● 선택 동의 | `서비스 내 프로필 표시` |
| **카카오계정(이메일)** | ● 선택 동의 [수집] | `계정 식별 및 로그인` |

> **이메일을 "선택 동의"로 해야 하는 이유:**
> - "필수 동의"로 하면 이메일 없는 카카오 계정이 로그인 못 함
> - "선택 동의"면 이메일이 있는 사용자만 제공, 없어도 로그인 가능
> - 이 항목은 **비즈 앱 전환 후**에만 설정 버튼이 활성화됨

### 8단계: REST API 키 + Client Secret 확인

1. 왼쪽 메뉴 → **앱 설정** → **플랫폼 키**
2. **REST API 키** 복사 (예: `1cf9ac566d7f142434857ddd9613f9fa`)
3. 왼쪽 메뉴 → **카카오 로그인** → **보안** (또는 **고급**)
4. **Client Secret** → "코드 발급" → 상태 "사용" → 코드 복사

### 9단계: Supabase에 카카오 연동

1. https://supabase.com/dashboard → 프로젝트 선택
2. 왼쪽 메뉴 → **Authentication** (🔐 아이콘)
3. CONFIGURATION 섹션 → **"Sign In / Providers"**
4. **Kakao** 찾아서 클릭
5. 설정:
   - **Kakao enabled**: ON
   - **REST API Key**: Kakao에서 복사한 REST API 키
   - **Client Secret Code**: Kakao에서 발급한 Client Secret
   - **Allow users without an email**: ON
6. **Save**

### 10단계: 코드 설정 (이미 완료)

`src/hooks/useAuth.ts`에서 카카오 로그인 시 scope 설정:
```typescript
const signInWithKakao = useCallback(async () => {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'kakao',
    options: {
      redirectTo: window.location.origin,
      scopes: 'profile_nickname profile_image',
    },
  });
  if (error) console.error('Kakao login error:', error);
}, []);
```

### 11단계: 테스트

1. `http://localhost:3001` 접속
2. 헤더 → "로그인" 클릭
3. "카카오로 시작하기" 클릭
4. 카카오 로그인 화면 → 동의 후 로그인
5. 온보딩 화면 표시: "환영합니다, {닉네임}님!"
6. 온보딩 완료 후 헤더에 프로필 아바타 표시

## 트러블슈팅

| 에러 | 원인 | 해결 |
|------|------|------|
| KOE205 (account_email) | 이메일 동의항목 미설정 | 비즈 앱 전환 → 이메일 선택 동의 설정 |
| KOE205 (비즈 앱 전환 불가) | 앱 아이콘 미등록 | 128px 아이콘 등록 후 비즈 앱 전환 |
| redirect_uri_mismatch | Redirect URI 불일치 | Kakao에 Supabase callback URL 정확히 입력 |
| 로그인 후 빈 화면 | Supabase Provider 미설정 | Sign In / Providers → Kakao enable |
