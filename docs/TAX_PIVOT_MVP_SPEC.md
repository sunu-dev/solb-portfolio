# 세무 비서 피봇 — MVP 설계 스펙 (v1)

> 출처: 12인 생존 패널(pivot-required 만장일치) → 세무 피봇 설계 워크플로(리서치 5 + 설계 5인 + 종합, 2026-06-02).
> 코어 = "다증권사 통합 해외주식 양도소득세 **계산·정리 도구**"
> ⚠️ 게이트 (2026-06-03 갱신 — 조건부): **v1 합산기(증권사 제공 세액 합산·정리)는 세무사 감수 불요**, 변호사 약관·카피만. **v2 자체계산(raw 재계산)만 세무사 감수+E&O 보험 필수.** 상세 §v1 합산기 결정 참조.

## 한 줄 정의

**여러 증권사에 흩어진 내 해외주식 손익을 한 곳에서 합쳐, 올해 예상 양도세와 250만원 공제 잔여를 보여주는 계산·정리 도구.** 토스(자사 거래만)·ChatGPT(내 보유 데이터 모름)가 구조적으로 못 주는 단 하나의 공백 = **다증권사 통합 손익통산**. 신고는 사용자가 홈택스에서 직접(셀프). 이 도구는 신고를 대신하지 않는다.

## 세법 사실 확정표 — verified vs ⚠️needs-confirm

### ✅ verified (taxRates.ts SSOT에 박제, 코드 단정 가능)
| 항목 | 확정 사실 |
|---|---|
| 세율 | 국외주식 양도차익 = 양도세 20% + 지방소득세 2% = **합계 22% 단일세율**(누진 아님) |
| 기본공제 | 연 **250만원**, **국내·국외 합산 후 1회**. 통산 모두 마친 뒤 적용(따로 250씩 빼면 과소납부) |
| 해외주식 간 통산 | 같은 과세기간 내 해외주식 차익·차손 상계 가능(국가 간 무관) |
| 손실 이월공제 | **불가**. 같은 과세연도 내만. **금투세 5년 이월은 폐지로 무효 — 이월 로직 절대 금지** |
| 국내↔해외 통산 | 2020.1.1+ 허용, 단 **'과세대상 국내주식'(대주주/비상장/장외)만**. 소액주주 장내는 비과세 → 통산 풀 제외 |
| 금투세 | **2024.12.10 폐지 확정**. 현행 양도소득세 체계로 설계 |
| 신고 | 예정신고 없음. 익년 5/1~5/31 확정신고 1회. 무신고가산세 20%(부정 40%) |
| 납세의무자 | 양도일까지 계속 5년 이상 국내 거주자만 국외주식 과세 |
| 대주주 | 종목당 50억원+(2025.9.15 확정). 코스피 1%/코스닥 2% |
| 증권거래세 2026 | 코스피 0.05%(+농특세=0.20%), 코스닥 0.20% |
| 세무사법 §2 독점 | 세무대리 = 신고대리·세무서류작성·기장대행·상담/자문. 무료라도 형사처벌 |
| 세무사법 §20③ | 2026.1.2 시행. 무자격자 세무대리 **오인 표시·광고 자체 금지**. 1년/1천만원 |
| 세무사법 §2의2 | 세무대리 소개·알선 **건당 수수료 금지**(정액 광고료만) |
| 합법 앵커 | 삼쩜삼 4년 무혐의 + 기재부 유권해석: **'사용자 직접 입력 + 자료수집·단순계산 한정'이면 세무대리 아님** |

### ⚠️ needs-confirm (화면에 'X — 세무사 확인 필요' 노출, 단정 금지)
환율 환산 기준일(체결일 vs 결제일 T+2) · lot 취득가 산정법(이동평균/FIFO, mergedAvgCost 근사 가능 여부) · 환차손익 처리 · 필요경비 범위 · 배당 분리과세(MVP 밖) · ISA 2026 한도(MVP 밖) · 2025 귀속 신고기한 5/31일요일→6/1 · **OCR 자동추출 = 삼쩜삼 '직접 입력'과 법적 동치 여부(핵심 쟁점)**.

## 법무 안전선

**세무사법 — 안전지대 단 하나**: '자료수집 + 단순 세액계산 한정, 최종 신고는 본인이 홈택스 셀프.' 코드·카피·약관 3층 동시 강제.
1. 신고 자동제출·대리 0줄. 산출물은 '홈택스 입력용 정리 명세'까지만.
2. **'세무판단 자문 0' 가드 신설** — AI는 계산 결과만, "이렇게 팔면 절세" 차단. 방향0 OVERRIDE와 동형으로 `alertCompliance.ts`에 `TAX_FORBIDDEN_PHRASES` + sanitize.
3. **카피 lint 확장(§20③)** — 금지어: 세금환급·신고대행·절세전문·기장대행·세무사·세무대리·세무상담·**세무 비서(오인 소지)**. 허용: 예상 양도세 계산기·해외주식 손익 정리·신고 준비 자료. 결제 페이지·버튼·도움말까지 lint.
4. **제휴 세무사 = MVP 제외.** 붙이면 건당 수수료 절대 금지, 정액 광고료만.
5. 민감정보(주민번호) 과수집 금지(자비스 8.5억 과징금 선례).

> ⚠️ **네이밍**: 세무 기능을 '해외주식 양도세 계산기'/'투자 세금 정리 도구'로 리네이밍(변호사 확인). '비서'가 세무 맥락에 결합되면 §20③ 오인광고 위험. 앱 제품명과 세무 기능 라벨 분리.

**§101 보존**: AI 촉·분석(투자판단 정보)은 영구 무료 → 면제 전제 보존. 세무는 별개 PRO지만 단일 묶음으로 packaging해 'AI 자문에 대가' 해석 차단. 약관·결제 화면 명문화.

**면책 4축**(약관 v5, legalVersions.ts 정합): ①계산·정리 도구이며 세무대리·상담·자문 아님 ②추정치, 신고 책임 본인 ③정확성은 입력 데이터 의존 ④최종 신고는 홈택스 셀프. ⚠️ 약관규제법 §7: 중과실 면책 무효 → **계산 정확성 자체를 세무사 1차 감수로 담보**.

## MVP v1 범위

**넣는다**: ①다증권사 통합 예상 양도세 보기(무료 Aha, '평단 추정치' 라벨 강제) ②what-if 매도 예상세액 카드 ③250만 공제 현황판(국내외 합산 1카운터) ④손익통산(같은 해만, 이월 UI 금지, 국내는 과세대상 분류 게이트) ⑤투자탭 상시 위젯(연1회→매일 정보 레이어) ⑥OCR 정산내역 입력(사용자 확인·편집 강제) ⑦컴플라이언스 골격.

**미룬다**: 신고 자동제출·홈택스 연동·세무사 연결(위반/불필요) · lot 자체 재계산(needs-confirm) · 국내 양도세·배당·ISA 계산 · PG 실결제 · 비-USD 다통화 · 세무 전용 상위 SKU.

## 데이터 아키텍처 (3계층)

- **Position (기존 유지)**: StockItem·mergedHoldings 그대로. 평단은 보유 관리용, **세무 입력값으로 직접 안 씀**. `StockItem.purchaseDate?` 옵셔널 추가만.
- **Transaction (신설, Supabase)**: `transactions(user_id, symbol, market, broker, type, trade_date, settle_date, shares, price_local, currency, fee_local, fx_rate, source)` + RLS self-only. 양도세 SSOT.
- **FX (신설)**: `fx_rates(date, base, quote, rate, source)` 일자별 공식 매매기준율. 결측일 직전영업일 carry-forward.
- **단일 환산 게이트** `toKrwAtSettle(amount, currency, settleDate)` — 모든 원화 환산이 이 함수만 통과. ⚠️ spot·1400 fallback이 세무 경로에 절대 못 들어오게 차단.
- **취득가**: lot별 (매수가×매수결제일환율+수수료). mergedAvgCost 세무 취득가로 안 씀. v1은 증권사 제공 취득가 신뢰·검증·정리 모드.
- **입력 우선순위**: 증권사 정산내역 PDF/CSV > 거래내역 CSV > 거래내역 OCR(기존 자산 확장=moat) > 수기.

## v1 합산기 결정 (2026-06-03 — "세무사 감수 꼭 필요?" 검토 결과)

3관점 워크플로(보수 변호사 + 실용 제품 + 선례 웹) 결론: **세무사 감수는 법적 의무가 아니라 calc-정확성 책임축(약관규제법 §7 중과실 면책 무효)의 리스크 완화 수단.** 필요성은 "무엇을 보여주느냐"가 결정한다.

**결정 트리**:
```
우리가 새 세액 "숫자"를 만들어내는가?
├─ NO (증권사 제공 양도세 내역 합산·정리만, 공식 출처 미러)  → ✅ 감수 불요 (v1)
└─ YES (raw 체결내역에서 취득가·세액 자체 재계산)            → 🔴 감수+E&O 보험 필수 (v2)
```

**결정적 발견 — needs-confirm 4개 중 2개는 국세청 공식 답이 있어 우리가 계산할 필요 없음** (⚠️ 코딩 전 1차 확인 권장이나, 합산기 경로는 이것에 의존하지도 않음):
- 환율 환산 기준일 = **결제일(T+2)** (국세청 국제세원-229, 2010.05.10) — 체결일 아님
- lot 취득가 = **증권사 제공 이동평균법** 사용 타당 (국세청) — FIFO/평단 자체 산정 불요
- **합계액 신고 허용** (국세청) — 종목별 정밀 재계산조차 불필요
- 환차손익·필요경비 = 회색지대 → **v1에서 안 건드림**
- 현실: 증권사들이 이미 HTS/웹에서 "양도소득세 계산내역"(이동평균 기반) 제공(신한·한국투자·삼성·유안타) → 합산기는 현실 데이터로 구현 가능.

**선례 만장일치**: Koinly·Sharesight·삼쩜삼·토스 누구도 "감수받고 출시"가 아님 — **추정 도구 + 면책 + 전문가 유도** 3종 세트. 단 삼쩜삼은 'TA(대리 성격)'로 넘어가자 세무사회 신규 고발 → "정리·추정"에 머물고 "대신 신고"로 가면 안 됨.

### v1 최소 Aha 슬라이스 (검증용 — 변호사는 공개·과금 직전 게이트, 카나리 검증엔 불요)

**한 화면**: "흩어진 N개 증권사를 합치니 올해 예상 양도세 ₩X, 250만 공제 중 Y 남음".

- **입력(최소)**: 증권사별 올해 실현손익(KRW) 1줄씩 — 증권사 "양도소득세 계산내역"에서 사용자가 수기 입력 또는 OCR. (증권사가 이미 이동평균·결제일환율로 KRW 환산 → 우리는 **환율 계산 안 함**.)
- **계산(단순 산수만 — 우리 판단 0)**: 합산손익 = Σ(증권사별 실현손익, 같은 과세연도·해외) / 250만 공제 잔여 = max(0, 2,500,000 − 합산손익) / 예상세액(**추정**) = max(0, 합산손익 − 2,500,000) × 22%.
- **출력**: 증권사별+총합 손익 / 250만 공제 막대(사용·잔여) / "예상 양도세 약 ₩X (추정)" / 신고시기(5월)·홈택스 링크 / "추정·신고책임 본인" framing 상단 고정.
- **DON'T**: 환율 계산, lot 산정, 환차·필요경비, 단정 숫자("1,847,000원"), "이 금액 신고하세요" 지시, 절세 행동 권유.
- **데이터 모델(v1 — 가벼움)**: `transactions`(lot)·`fx_rates`·`toKrwAtSettle`은 **v2 자체계산용**. v1은 증권사별 실현손익 summary만 → 신규 `tax_broker_summary { user_id, tax_year, broker, realized_gain_krw, source, captured_at }` (검증 단계는 클라이언트 state로도 충분). gateTaxAdvice·TAX_FORBIDDEN_PHRASES·taxRates SSOT(2026-06-03 골격)는 그대로 재사용.
- **잔존 리스크 = 엔지니어링으로**: 우리가 하는 유일한 계산(다증권사 합산·통화 혼입)에 **결정론적 단위테스트 + 통화 가드**.
