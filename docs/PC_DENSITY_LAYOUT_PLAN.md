# PC 정보밀도·와이드 레이아웃 설계안 (A: 설계 → 합의 → 구현)

> 2026-06-20. 파운더 의도 재확인: 토스블루(색)가 아니라 **"화면을 효율적으로 꽉 채우는, 깔끔·모던한 레이아웃"** = 정보밀도·와이드 공간 활용.
> 제약(유지): `feedback_design_direction`(토스/카뱅풍 미니멀, Bloomberg 잡탕 NO) · `feedback_balance`(균형·여백) · `feedback_pc_mobile`(모바일 단일컬럼 보존) · 방향0·매매없음.
> 원칙: **꽉 채운다 = 빈 여백 낭비를 줄이고 와이드에서 멀티컬럼으로 밀도↑, 톤은 깔끔 유지.** 모든 변경은 lg(1024)/xl(1280)+ 한정, 모바일/태블릿 불변.

## 0. 근본 원인 (모든 화면 공통)
| # | 현재 | 문제 | 위치 |
|---|---|---|---|
| R1 | 본문 컨테이너 `maxWidth:1200px` 고정 | 1920에서 좌우 ~700px 빔 ("모바일 늘린 화면") | `page.tsx:270` |
| R2 | 카드 `maxWidth:480` 캡(Insights·Events) | 900px 본문 안에서도 카드가 480 → 또 절반 빔 | `InsightsSection:208`·`EventsSection:724` |
| R3 | News·Insights·Events 본문 단일 컬럼 | 와이드에서 카드가 세로로만 길게 | 각 섹션 |
| R4 | 보유 테이블 6컬럼 고정(160/90/90/120/120/60) | 와이드 우측 dead space | `PortfolioSection:770` |

→ **R1·R2가 루트.** 컨테이너 캡을 단계 확장 + 카드 캡 해제 + 멀티컬럼이 핵심.

## ★ 설계 표준 (디자인 템플릿) — 들어가기 전 못박음 (2026-06-20 구현)
화면마다 일관되고 **간격이 밀리거나 삐져나오지 않게** 하는 공통 규칙. 코드 SSOT = `globals.css`(PC 정보밀도 표준 섹션) + `designTokens.ts`(breakpoint/container/gridGap).

**브레이크포인트** (Tailwind v4 = 우리 기준): `md 768 · lg 1024 · xl 1280 · 2xl 1536`. 밀도 변경은 **lg/xl+만**, 모바일/태블릿 불변.

**컨테이너 폭** (`.app-shell`): `≤lg 1200 · xl 1360 · 2xl 1520`. (page.tsx 컨테이너에 적용)

**폰트 (디자인 템플릿)**: `T.fontSize`(xs10~display20) **고정 스케일 유지**. ⚠️ **밀도를 위해 글자를 줄이지 않는다** — 좁아 보이면 컬럼을 늘려 채우지, 폰트를 줄여 욱여넣지 않음(가독성·접근성). 와이드에서 섹션 제목만 선택적 +1~2px(clamp) 허용, 본문/숫자/라벨은 불변.

**간격 (gap/리듬)**: 멀티컬럼 gap = **lg 20 / xl 24**(`.fill-grid`·`.cols-2-xl`). 카드 내부 패딩·라운드 **불변**(아기자기 유지). 수직 리듬 기존 유지.

**그리드 SSOT (재사용 클래스)**:
- `.fill-grid` — 카드 멀티컬럼(`--fill-min`으로 카드 최소폭 지정, lg부터 auto-fill). Insights·Events용.
- `.cols-2-xl` — 순서 보존 2열(xl+). News용.
- `.show-from-xl` — 와이드 전용 보조 컬럼/정보(테이블).
- `.truncate-1` — 단일행 말줄임.

**오버플로 방지 표준 (삐져나옴 0)**:
1. grid는 `minmax(0, 1fr)` (그냥 `1fr` 금지 — 내용이 컬럼을 밀어냄).
2. 채움 카드는 `minmax(min(100%, X), 1fr)` (좁은 폭에서 카드가 컨테이너보다 커지는 것 차단).
3. grid/flex 자식은 `min-width: 0`(`.fill-grid > *`에 기본 적용 — flex/grid의 #1 오버플로 원인).
4. 미디어/차트/이미지 `max-width: 100%`.
5. 텍스트 — 단일행 ellipsis(`.truncate-1`) / 다행 `overflow-wrap` / 한국어 `word-break: keep-all`(전역 적용됨).

## 1. 전역 셸 — 반응형 컨테이너 (foundation) ✅ 구현 완료 (2026-06-20)
`.app-shell` 클래스(globals.css) + page.tsx 컨테이너 적용. xl 1360 / 2xl 1520 단계 확장. margin auto 유지. **이 단계만으론 넓은 단일컬럼** → §2~5의 멀티컬럼이 공간을 실제로 채움.
- 본문 컨테이너 폭을 **단계형**으로: `≤lg 1200(현행) · xl(1280+) 1360 · 2xl(1536+) 1520`. 우측 사이드바 280 유지 → 콘텐츠 영역 xl에서 ~1040, 2xl ~1200 확보.
- 좌우 패딩은 유지(여백 균형). **밀도는 "더 넓게"가 아니라 "넓어진 공간을 멀티컬럼으로"** 채워서 올림(아래 각 섹션).
- 모바일/태블릿(<lg) 완전 불변.
- 효과/위험: 셸 1곳 수정으로 전 화면 공간 확보. 위험 낮음(폭만). 단 각 섹션이 멀티컬럼을 안 받으면 그냥 더 넓은 단일컬럼이 되므로 §2~5와 세트.

## 2. AI 인사이트 (Insights) — 가장 빈 화면
- **현재**: AI촉·멘토·스토리 카드가 `maxWidth:480` 단일 컬럼 → 와이드에서 좌측 480만 차고 나머지 빔.
- **제안**: 카드 480 캡 해제 → **반응형 카드 그리드**(`lg 2열 · xl 3열`, gap 20, `repeat(auto-fill, minmax(320px,1fr))`). AI촉 3카드는 한 줄로, 멘토/스토리도 그리드.
- 톤: 카드 자체 디자인 불변(아기자기 유지), 배치만 그리드. 카드 사이 gap 넉넉히 → 깔끔·밀도 양립.
- 효과 高 / 위험 中(섹션 내부 구조 손봄, 카드 폭 가변 대응 확인 필요).

## 3. 뉴스 (News) — 단일 컬럼 → 멀티컬럼
- **현재**: 뉴스 리스트 단일 컬럼.
- **제안**: `xl+ 2열`(시간순 보존 위해 CSS columns 아닌 명시 row-flow 2-col 그리드). 카드 폭 ~480 유지하되 2열로 채움.
- 톤·카드 불변. 모바일 단일.
- 효과 中 / 위험 低.

## 4. 이벤트 분석 (Events) — 카드 캡 해제 + 멀티컬럼
- **현재**: 실적/배당 카드 `maxWidth:480` 중앙, 내부 3-col 그리드는 이미 있음.
- **제안**: 카드 480 캡 해제, **xl 2열 카드 그리드**. 내부 미니 그리드는 유지.
- 효과 中 / 위험 低.

## 5. 포트폴리오 (Portfolio) — 테이블·위젯 밀도
- **보유 테이블**: 와이드(xl+)에서 보조 컬럼 **점진 노출** — `비중%`(파생, descriptive) 우선, 선택적으로 `일간변동`·`섹터`. `.hide-below-xl` 패턴으로 모바일/좁은 화면 숨김. dead space → 정보로.
- **위젯 그리드**: 현재 2열(`portfolio-widgets-grid` 1fr 1fr) → **xl 3열**로(헬스·섹터갭·트레이드미러 등 더 조밀).
- **상단 요약(Dashboard)**: 이미 2열. xl에서 핵심 지표를 한 줄 더 노출(환차손익·목표달성률 등) 검토.
- 효과 高 / 위험 中(테이블 컬럼 = `feedback_design_consistency` 정렬·균형 체크 필수).

## 6. 종목 상세 (AnalysisPanel) — 풀 2컬럼 (★ 핵심, 그간 미룬 것)
- **현재**: 700→880(lg)→1080(opt-in) 중앙 모달, 세로 스크롤(차트→기술→멘토→뉴스→시뮬).
- **제안**: lg+ **2컬럼** — **좌: 차트(+기술지표) sticky / 우: Price·AI분석·멘토·뉴스·시뮬 스크롤**. 모달 폭 ~1100.
  - 구현 안전책: 차트 블록(1303~1512 중첩 fragment)을 **명시 추출**해 좌측 컬럼 컴포넌트로(또는 wrapper div + CSS grid), 모바일/기본은 현행 단일 유지.
  - ⚠️ 이 부분만 **프리뷰 시각 검증 루프 필수**(blind 배포 금지). 나머지 §1~5는 build로 충분.
- 효과 最 / 위험 高(구조 재배치 + 시각검증).
