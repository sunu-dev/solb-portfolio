## 1부: 현재 스크린샷 vs 토스증권 비교 진단

### UI 전문가

> **현재 우리 앱의 문제를 하나하나 짚겠습니다.**
>
> #### 문제 1: 매크로 카드 — "비어 보임"
> | 항목 | 우리 | 토스증권 |
> |------|------|---------|
> | 카드 크기 | 가로 전체를 6등분, 빈 공간 과다 | 콘텐츠에 꽉 맞는 크기, 또는 가로 스크롤 |
> | 카드 높이 | 너무 높음, 콘텐츠 대비 여백 과다 | 콤팩트하게 라벨+숫자+변동률만 |
> | 시각적 무게 | 테두리만 있고 내용이 없어 보임 | 숫자가 크고 눈에 들어옴 |
> | 데이터 없을 때 | "--"만 표시, 카드 사이즈는 동일 | 로딩 스켈레톤 or 숨김 |
>
> **토스 해법:** 매크로는 상단에 가로 스크롤 스트립으로, 각 아이템은 label+value+change 한 줄로. 카드가 아닌 인라인 텍스트.
>
> #### 문제 2: 종목 리스트 — "밀도 부족"
> | 항목 | 우리 | 토스증권 |
> |------|------|---------|
> | 행 높이 | 80px+ (아바타 포함) | 60px (콤팩트) |
> | 왼쪽 정보 | 회색 아바타 원 + 티커 + 한글명 | 종목 로고 이미지 + 종목명(한글 우선) + 현재가 |
> | 오른쪽 정보 | 가격 + 등락률 | 보유수량 + 평가금 or 등락률 |
> | 구분선 | 전체 너비 border | 양쪽 패딩 제외한 부분 divider |
> | 호버 효과 | 배경 약간 변경 | 매끄러운 터치 피드백 |
>
> **토스 해법:** 한글 종목명을 먼저, 티커를 보조로. 로고 없으면 컬러 아바타. 행 높이를 60px로 줄이고 밀도 높이기.
>
> #### 문제 3: 전체 레이아웃 — "구조가 없음"
> | 항목 | 우리 | 토스증권 |
> |------|------|---------|
> | 상단 영역 | 헤더 바로 아래 매크로 나열 | 총 자산 요약 카드 (큰 숫자) |
> | 시각적 앵커 | 없음 | 총 평가 금액이 시각적 중심 |
> | 섹션 구분 | 그냥 위에서 아래로 나열 | 카드 그룹핑, 섹션 제목, 여백으로 구분 |
>
> **토스 해법:** 최상단에 "총 평가금" 히어로 섹션. 큰 숫자가 시선을 잡음.

### UX 전문가

> **근본 문제: "토스의 색상"을 가져왔지만 "토스의 구조"를 가져오지 않았습니다.**
>
> 토스증권 앱을 열면 첫 화면에 보이는 것:
> ```
> ┌─────────────────────────────┐
> │  내 투자                    │  ← 섹션 제목
> │                             │
> │  총 평가금액                 │  ← 라벨 (13px, 회색)
> │  ₩12,450,000               │  ← 큰 숫자 (28px, 굵게)
> │  +₩1,230,000 (+10.9%)      │  ← 수익/손실 (빨강/파랑)
> │                             │
> │  ┌─────┐ ┌─────┐ ┌─────┐  │
> │  │ 주문 │ │ 이체 │ │ 혜택 │  │  ← 퀵 액션 버튼
> │  └─────┘ └─────┘ └─────┘  │
> ├─────────────────────────────┤
> │  보유 종목                   │  ← 섹션 제목
> │                             │
> │  [로고] 마이크론       $420  │  ← 한글명 우선
> │         MU         +2.3%   │  ← 티커는 보조
> │  ─────────────────────────  │  ← 미세 구분선
> │  [로고] 마이크로소프트  $385  │
> │         MSFT        -0.5%  │
> │  ─────────────────────────  │
> │  [로고] 브로드컴      $310  │
> │         AVGO        +1.2%  │
> └─────────────────────────────┘
> ```
>
> **핵심 차이:**
> 1. **"총 평가금"이 첫 번째로 보임** — 우리는 이게 없음
> 2. **한글 종목명이 주역** — 우리는 티커(ASTX)가 주역
> 3. **숫자 크기 대비** — 큰 숫자(가격)와 작은 숫자(라벨)의 차이가 극명
> 4. **빈 공간이 의도적** — 우리는 빈 공간이 "빼먹은" 느낌

## 2부: 구체적 수정 사항 합의

### 전원 합의 — 컴포넌트별 수정안

#### 1. 포트폴리오 히어로 (신규 추가)
> 최상단에 포트폴리오 요약 카드를 추가합니다.
> ```
> ┌──────────────────────────────┐
> │  내 투자                      │
> │                              │
> │  총 평가금액                   │  12px, #8B95A1
> │  $12,450.00                  │  28px, font-bold, #191F28
> │  +$1,230 (+10.9%)            │  14px, font-semibold, #EF4452
> │                              │
> │  투자금 $11,220  |  수익 $1,230│  12px, #8B95A1
> └──────────────────────────────┘
> ```
> - 배경: 흰색 카드, border-radius 16px
> - 이 카드가 페이지의 **시각적 앵커** 역할

#### 2. 매크로 스트립 수정
> 카드 형태 → 인라인 가로 스크롤로 변경
> ```
> S&P 500 5,667.8 ▲0.12%  │  NASDAQ 17,890 ▼0.34%  │  USD/KRW 1,501 ...
> ```
> - 높이: 44px 한 줄
> - 배경: 투명 (카드 없음)
> - 각 항목: label(11px) + value(14px, bold) + change(11px)
> - 가로 구분: 미세한 vertical divider
> - 오버플로: 가로 스크롤

#### 3. 종목 리스트 수정
> - **한글명을 주역으로** (마이크론, 마이크로소프트)
> - 티커(MU, MSFT)는 보조 텍스트로
> - 아바타: 종목 첫 글자 → 컬러 배경 (종목마다 다른 색상)
> - 행 높이: 64px
> - 가격: 오른쪽 정렬, 15px bold
> - 등락률: 가격 아래, 12px
> ```
> [M] 마이크론          $420.00
>     MU                +2.34%
> ```

#### 4. 뉴스 카드 수정
> - 현재: 제목 + 요약 + 출처가 나열됨 → 너무 길음
> - 수정: 제목만 (한 줄), 출처 + 시간 (보조)
> - 카드 없이 리스트 형태로
> ```
> [시장]  월가 기술주 이탈 본격화...나스닥 1.6% 급락
>         포커스온경제 · 2시간 전
> ──────────────────────────────
> [AI]   연말 '조용한 조정' 속 AI 대형주 흔들림
>         mstoday.co.kr · 3시간 전
> ```

#### 5. 이벤트 테이블 수정
> - 테이블 → 카드 리스트로 변경
> - 각 종목을 개별 카드로:
> ```
> ┌──────────────────────┐
> │  MU 마이크론          │
> │  기준 $487 → 현재 $422│
> │  ▼ -13.2%   최대 -27%│
> └──────────────────────┘
> ```

## 3부: 근본 원인과 프로세스 개선

### PM

> **왜 계속 토스와 다른 결과가 나오는가?**
>
> 1. **레퍼런스 없이 코딩함** — 토스 실제 화면을 보면서 만들어야 하는데, 텍스트 설명만으로 구현함
> 2. **색상 = 디자인이라고 착각** — 색상 토큰만 맞추면 된다고 생각. 실제로는 비율, 밀도, 타이포 스케일, 시각적 무게가 핵심
> 3. **데스크톱 너비 문제** — 토스는 모바일 우선. max-width 480px 정도가 토스 느낌. 넓게 펼치면 어떤 디자인이든 빈약해 보임
> 4. **"총 평가금" 히어로가 없음** — 토스의 첫 화면에서 가장 큰 요소인데 우리는 빼먹음

### UI 전문가

> **해결 방향:**
>
> 1. **모바일 퍼스트**: max-width 480px 중앙 정렬. 데스크톱에서는 폰 화면처럼 보이게
> 2. **포트폴리오 히어로 추가**: 총 평가금 큰 숫자가 첫 번째로 보이게
> 3. **종목명 한글 우선**: "마이크론" 크게, "MU" 작게
> 4. **매크로 인라인화**: 카드 제거, 가로 한 줄 텍스트
> 5. **밀도 높이기**: 행 높이 줄이기, 불필요한 패딩 제거
> 6. **뉴스 간소화**: 제목 + 출처만, 카드 없이 리스트

## 최종 합의: 즉시 실행 사항

| 순서 | 작업 | 효과 |
|------|------|------|
| 1 | max-width 480px + 모바일 우선 | 토스 비율과 일치 |
| 2 | 포트폴리오 히어로 카드 추가 (총 평가금) | 시각적 앵커 확보 |
| 3 | 매크로 카드 → 인라인 스트립 | 공간 절약 + 밀도 |
| 4 | 종목: 한글명 주역, 컬러 아바타, 행 높이 64px | 토스 종목 행과 일치 |
| 5 | 뉴스: 리스트 형태, 제목+출처만 | 간결함 |
| 6 | 이벤트: 테이블 → 카드 리스트 | 모바일 친화 |
