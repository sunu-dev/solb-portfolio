## 1부: 증권사/금융 서비스의 실시간 시세 구조

### FE 전문가

> **주식 시세 업데이트 방식 3가지:**
>
> | 방식 | 설명 | 지연 | 누가 쓰나 |
> |------|------|------|----------|
> | **WebSocket** | 서버가 가격 변할 때 즉시 push | 0.1~1초 | 토스증권, 키움, 미래에셋 (실시간 HTS/MTS) |
> | **Polling** | 클라이언트가 N초마다 API 호출 | N초 | 야후 파이낸스, 일반 금융 앱 |
> | **SSE (Server-Sent Events)** | 서버가 단방향 push | 1~5초 | 일부 대시보드 |
>
> **토스증권의 구조:**
> ```
> 토스증권 앱
>   ↕ WebSocket 연결 (상시)
> 토스 서버
>   ↕ 증권사 데이터피드 (KRX/NASDAQ)
> 거래소 (한국거래소, NYSE, NASDAQ)
> ```
> - 거래소에서 실시간 체결 데이터를 받아서
> - 토스 서버에서 WebSocket으로 앱에 push
> - 가격이 변할 때마다 즉시 업데이트
>
> **야후 파이낸스의 구조:**
> ```
> 야후 파이낸스 웹
>   → 5초마다 REST API 호출 (Polling)
> 야후 서버
>   → 지연 시세 (15분~실시간)
> ```

### PM

> **우리의 현재 구조:**
> ```
> SOLB 앱
>   → 30초마다 REST API 호출 (Polling)
> Finnhub API / Yahoo Finance
>   → 실시간~15분 지연 시세
> ```
>
> **문제: 30초면 충분한가?**
>
> | 사용자 유형 | 필요한 갱신 주기 | 이유 |
> |-----------|---------------|------|
> | 데이 트레이더 | 1초 이하 | 초 단위로 매매 |
> | 스윙 트레이더 | 5~15초 | 분 단위 판단 |
> | **초보 장기 투자자 (우리 타겟)** | **30초~1분** | **하루 1~2번 확인** |
> | 관망 | 1시간+ | 가끔 확인 |
>
> **결론: 우리 타겟(초보 장기 투자자)에게 30초 polling은 충분합니다.**

## 2부: 그래도 개선할 수 있는 것

### 백엔드 전문가

> **단계별 실시간 개선안:**
>
> #### Level 1: 현재 (30초 Polling) — $0
> ```
> 30초마다 → Finnhub REST → 가격 업데이트
> ```
> - 장점: 간단, 비용 없음
> - 단점: 30초 지연, API 호출 수 많음
>
> #### Level 2: Finnhub WebSocket — $0 (무료 플랜 포함)
> ```
> WebSocket 연결 유지 → 체결 시 즉시 push → 가격 업데이트
> ```
> - **Finnhub 무료 플랜에 WebSocket 포함!**
> - 미국 주식 실시간 체결가 제공
> - 연결 1개, 최대 50개 종목 구독
> - 장점: 진짜 실시간 (1초 이내), API 호출 수 줄어듦
> - 단점: 클라이언트에서 직접 연결해야 함 (서버리스 환경에서 제약)
>
> #### Level 3: 서버 중계 WebSocket — Phase 3
> ```
> SOLB 서버 (WebSocket 서버)
>   ↕ Finnhub WebSocket
>   ↕ 사용자 브라우저 (WebSocket)
> ```
> - 서버가 중계해서 모든 사용자에게 push
> - 장점: 사용자 수 제한 없음
> - 단점: 서버 비용 발생

### 주식 고수

> **Finnhub WebSocket은 무료인데 왜 안 쓰나요?**
>
> 맞습니다. Finnhub 무료 플랜에 WebSocket이 포함되어 있습니다.
> 브라우저에서 직접 연결하면 **$0으로 실시간 시세**를 받을 수 있습니다.
>
> ```javascript
> const socket = new WebSocket('wss://ws.finnhub.io?token=YOUR_API_KEY');
> socket.send(JSON.stringify({ type: 'subscribe', symbol: 'MU' }));
> socket.onmessage = (event) => {
>   const data = JSON.parse(event.data);
>   // data.data[0].p = 최신 체결가
>   // data.data[0].v = 체결량
> };
> ```
>
> **제한사항:**
> - 무료: 최대 50개 종목 동시 구독
> - 미국 주식만 (한국 주식은 안 됨)
> - 장 중(한국시간 23:30~06:00)에만 데이터 수신
> - 장 외 시간에는 마지막 체결가 유지
