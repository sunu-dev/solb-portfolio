## 2부: 해결 방안

### FE 전문가

> #### 방안 1: 가격과 캔들 분리 (가장 효과적)
>
> **현재:** 가격 → 캔들 → 화면 표시 (직렬, 5초)
> **개선:** 가격 먼저 표시 → 캔들은 백그라운드 (1초에 가격 보임)
>
> ```
> 병렬 시작:
>   ├── fetchMacro() (매크로 6개 병렬)     ─→ 1초
>   └── fetchAllQuotes() (종목 11개 병렬)  ─→ 1초
>   ↓ 여기서 화면 표시! (1초)
>   └── fetchAllCandles() (백그라운드, 순차) ─→ 3초 (화면 이미 보이는 중)
> ```
>
> **효과: 화면 표시까지 1초!** (캔들은 나중에 채워짐)
>
> #### 방안 2: 캔들 병렬 호출
>
> **현재:** 순차 + delay(150ms)
> **개선:** 3~4개씩 병렬 (Finnhub rate limit 60/분 고려)
>
> ```
> 현재: A → wait → B → wait → C → ... (순차 4초)
> 개선: [A,B,C] → wait → [D,E,F] → wait → ... (병렬 1.5초)
> ```
>
> #### 방안 3: 캔들 캐시 활용
>
> 캔들 데이터는 하루 1번만 바뀜 (일봉). localStorage에 캐싱하면 재방문 시 즉시 표시.
>
> ```
> 첫 방문: API 호출 (5초)
> 재방문: 캐시 즉시 표시 (0초) → 백그라운드 갱신
> ```
